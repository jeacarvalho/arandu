Demonstração: Contexto Biológico sem JavaScript

Problema Atual

Atualmente, o contexto biológico é carregado via JavaScript (fetch), o que causa:

  • ❌ Erros na console do navegador
  • ❌ Atraso no carregamento (tempo de fetch + renderização)
  • ❌ Dependência de JavaScript funcionando corretamente
  • ❌ Conteúdo invisível para crawlers (SEO)

Solução Proposta

Carregar o contexto biológico diretamente no servidor, assim como os outros dados do paciente:

// NO HANDLER DO PACIENTE (patient_handler.go)
func (h *PatientHandler) Show(w http.ResponseWriter, r *http.Request) {
  // 1. Carregar dados básicos do paciente
  patient, err := h.patientService.GetPatientByID(r.Context(), id)
  
  // 2. Carregar temas recorrentes
  themes, err := h.patientService.GetThemeFrequency(r.Context(), id, 10)
  
  // 3. Carregar contexto biológico (AGORA NO SERVIDOR!)
  medications, _ := h.biopsychosocialService.GetMedications(id)
  latestVitals, _ := h.biopsychosocialService.GetLatestVitals(id)
  avgVitals, _ := h.biopsychosocialService.GetAverageVitals(id, 30)
  
  // 4. Renderizar TUDO de uma vez
  detail := patientComponents.PatientDetail(patientDetail, sessionItems, bioContext)
}
// NO TEMPLATE (detail.templ)
templ PatientDetail(patient PatientDetailItem, sessions []SessionItem, bioContext *BiopsychosocialPanelViewModel) {
  <div class="grid-2x2-card grid-card-1">
    <h2>Contexto Biológico</h2>
    @BiopsychosocialPanel(*bioContext, true)
  </div>
}

Benefícios

  • ✅ Sem JavaScript necessário para carregamento inicial
  • ✅ Carregamento instantâneo (já vem no HTML)
  • ✅ Sem erros na console (não há fetch falhando)
  • ✅ Melhor SEO (conteúdo visível para crawlers)
  • ✅ Experiência mais robusta (funciona mesmo com JS desabilitado)
  • ✅ Código mais simples (menos complexidade no frontend)

Próximos Passos

  1. Corrigir erros de compilação do Go (problemas de importação)
  2. Atualizar handler do paciente para carregar contexto biológico
  3. Atualizar template para renderizar contexto diretamente
  4. Manter funcionalidade de edição via modal (ainda com JavaScript)

Conclusão: A abordagem atual (carregar via JavaScript) é desnecessariamente complexa. Podemos simplificar drasticamente carregando tudo no servidor, como já fazemos com os outros dados do paciente.
